# AI Work Ethic

- Never leave TODO comments, placeholder code, or stub implementations. If you write it, finish it.
- Do not take shortcuts or skip over complexity. If something is hard, work through it fully.
- If you hit a blocker or a broader issue that's outside the immediate task scope, spin up a subagent to resolve it before continuing with the main task.
- Do not silently skip requirements. If a requirement can't be met, explain why — don't just omit it.
- Every file you create or modify should be complete and functional, not a skeleton to be filled in later.
